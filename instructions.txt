# Projekt-Spezifikation — Asian Superstore Website (vollständig & ausführlich)

## Kurzbeschreibung
Erstelle eine moderne, responsive und elegante Informations-Website für den Supermarkt **„Asian Superstore“**.  
Ziel: ansprechende Präsentation der Marke, Produktkategorien, wöchentliche Angebote und Filialen – plus ein einfaches Admin-System, mit dem der Ladenbesitzer Änderungen global und dauerhaft durchführen kann.

Wichtig: Die Seite ist **kein Onlineshop**. Sie soll auf **Netlify** oder **GitHub Pages** deploybar sein. Änderungen durch das Admin-Panel müssen **global** gespeichert werden — wir nutzen dafür **Netlify CMS**, das Änderungen direkt in das GitHub-Repository committet und die Seite automatisch neu deployed.

Sprache: **Deutsch**. Ton: freundlich, professionell, frisch, multikulturell.  
Slogan: **„Bringt die Welt zusammen.“**

## 1. Design & Branding (ausführlich)
Ziel: Premium / frisch / organisch / Halal → nicht schwarz-weiß.

### Farbpalette (Beispielwerte; Cursor/Designer kann leichte Varianten wählen)
- Primärgrün (frisch): #2E8B57 (oder #27A96B)  
- Sekundärgrün (Akzente): #8FD3A3  
- Creme / Natur: #FFF8F0  
- Warmes Beige: #E9DCC9  
- Dunkles Anthrazit (Text): #222222  
- Akzentfarbe (CTA / Highlights): #FFB64D (warmes Gold/Orange)

> Hinweis: Verwende die Primärfarbe für Buttons/Links/Hero-Highlights, Sekundärfarbe für Hover/Badges, Creme/Beige als Hintergrundsektionen. Text standardmäßig in Anthrazit. Keine reine Schwarz-Weiß-Ästhetik.

### Typografie
- Überschriften: Poppins oder Inter (stark, modern)  
- Fließtext: Inter oder system-fallback sans-serif  
- Gewicht: H1 = 700, H2 = 600, Body = 400  
- Zeilenhöhe großzügig für Lesbarkeit (1.4–1.6)

### Bildsprache
- Frisches Obst/Gemüse, asiatische Gewürze, Reis, Tees. Keine Fotos vom echten Laden nötig (Platzhalter/Stock-Bilder).
- Icons für Kategorien (lineare Icons, rundes Icon-Frame).

### UI-Elemente
- Große Hero-Sektion (Slogan + CTA "Angebote ansehen")  
- Category Cards (Bild/Icon + kurzer Text + Link)  
- Offer Cards (Preis, kurzer Beschreibungstext, Badge „Angebot“)  
- Saubere, minimale Navbar + Footer mit Kontaktinfos  
- Responsive Grid für Produkte & Angebote

## 3. Daten & Content (Format & Speicherort)

### Hauptdatenquellen
- `data/categories.json` — Liste der Kategorien (id, name, slug, short_description, image, icon)
- `data/offers.json` — Wöchentliche Angebote (id, title, description, price, old_price, valid_until, image, category_id)

**Wichtig:** Obwohl CMS die primäre Bearbeitungsoberfläche ist, sollen `categories.json` und `offers.json` auch im Repo existieren, damit Cursor lokale Platzhalter laden kann.

## 4. Admin & Global Storage — Netlify CMS (ausführlich)
Netlify CMS wird als statisches Admin-Interface installiert. Änderungen werden per Git commit & push in dein GitHub-Repository geschrieben und Netlify deployed automatisch die Seite.

### Vorteile
- Keine eigene Serverinfrastruktur
- Authentifizierter Admin-Login (Netlify Identity)
- Änderungen sind global und sofort sichtbar nach Deploy
- Einfaches Rollback via Git

### Benötigte Schritte (für README / Setup)
1. GitHub Repository erstellen (privat/öffentlich).  
2. Netlify Account erstellen und mit GitHub verbinden.  
3. Netlify Identity aktivieren (für Login).  
4. Netlify CMS konfigurieren:
   - `static/admin/config.yml` anlegen mit Collections für `categories` und `offers`.
   - Collections schreiben Dateien z. B. als JSON oder Markdown im Repo (`data/categories/*.json` oder `content/offers/*.md`).
5. Netlify-Build-Settings: `npm run build` (Vite-Build) als Build-Befehl.
6. Deploy durchführen — Admin erreichbar unter `/admin`.

### Beispiel: Collections (konzeptionell)
- Collection: Categories
  - Folder: `data/categories`
  - Format: JSON (oder individual JSON files pro Kategorie)
  - Fields: id, name, slug, short_description, image, icon

- Collection: Offers
  - Folder: `data/offers`
  - Format: JSON (oder Markdown)
  - Fields: id, title, description, price, old_price, valid_until, image, category

### Sicherheit & Rechte
- Netlify Identity verwaltet User/Passwort. Erstelle mindestens einen Admin-User.
- Optional: Git Gateway (Netlify) für direkten Commit in GitHub.
- Hinweise im README zum sicheren Umgang mit Credentials.

---

## 5. Fallback & lokale Admin-Variante (optional)
Falls du Netlify CMS nicht sofort einrichten willst, implementiere eine einfache clientseitige Admin-Seite als Interimslösung:

- `/admin-local` mit einfachem Login (hart codiertes Passwort in `env` während Entwicklung).
- Lädt `categories.json` und `offers.json` in Memory.
- Ermöglicht Add/Edit/Delete.
- Bietet JSON-Export (Download file) und JSON-Import (Upload file).
- **Wichtig:** Dokumentiere klar, dass diese Variante **nur lokal** wirkt — Änderungen müssen per Upload in das Repo (oder per Netlify UI) übernommen werden, damit sie global werden.
